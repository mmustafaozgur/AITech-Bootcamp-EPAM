# Memory Banks — Task Board Lab

This directory contains living knowledge-base files for AI assistants working on the **task-board-lab** project. Unlike the `specs/` directory (which holds product-level requirements and decisions), `memory-banks/` holds implementation-level knowledge that helps AI assistants produce consistent, high-quality code.

## Structure

```
memory-banks/
├── README.md                     # This file
├── architecture/
│   └── overview.md               # System architecture, patterns, and tech stack
├── conventions/
│   └── coding-standards.md       # Naming conventions, file structure, code rules
├── domain/
│   └── glossary.md               # Domain terminology and concept definitions
├── roles/                        # (reserved for role-specific guidance)
└── workflows/
    └── development-process.md    # Step-by-step implementation workflow
```

## How to Use

- Read relevant files before implementing a feature.
- Update files when conventions or architecture decisions change.
- Do **not** store product requirements here — use `specs/` for those.
- Do **not** store temporary notes here — these are persistent reference documents.
